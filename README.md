# 1. Introdução

O presente trabalho tem como objetivo desenvolver um repositório profissional utilizando Git e GitHub, aplicando conceitos de versionamento, organização de projetos, documentação e integração com plataformas profissionais.

A proposta do projeto PortfolioHUB é criar um ambiente organizado para armazenar projetos acadêmicos e pessoais, permitindo o compartilhamento de códigos, evolução de versões e apresentação profissional dos trabalhos desenvolvidos ao longo da graduação.

Além disso, o projeto demonstra o uso de ferramentas amplamente utilizadas no mercado de tecnologia, como GitHub Pages, GitHub e LinkedIn, fortalecendo o perfil profissional do estudante e ampliando sua presença digital.

---

# 2. Objetivos

## 2.1 Objetivo Geral

Desenvolver um repositório versionado no GitHub contendo projetos acadêmicos e pessoais, aplicando boas práticas de organização, documentação e compartilhamento profissional.

## 2.2 Objetivos Específicos

* Criar e configurar um repositório no GitHub;
* Utilizar Git para controle de versão;
* Organizar os projetos em pastas estruturadas;
* Desenvolver um README.md profissional;
* Publicar uma página utilizando GitHub Pages;
* Integrar o repositório ao LinkedIn;
* Demonstrar conhecimentos básicos de versionamento e hospedagem de projetos.

## Git

# 5. Criação e Versionamento do Projeto

Inicialmente foi criado um projeto simples de página web utilizando HTML e CSS.

Após a criação dos arquivos, o versionamento foi realizado utilizando comandos Git.

## Principais comandos utilizados

```bash
git init
```

Inicializa o repositório Git local.

```bash
git add .
```

Adiciona todos os arquivos para controle de versão.

```bash
git commit -m "Versão inicial do projeto"
```

Cria um ponto de salvamento do projeto.

```bash
git branch -M main
```

Define a branch principal.

```bash
git remote add origin https://github.com/seuusuario/PortfolioHUB.git
```

Conecta o repositório local ao GitHub.

```bash
git push -u origin main
```

Envia os arquivos para o GitHub.
